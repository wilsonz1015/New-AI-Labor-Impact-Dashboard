Methodology

This dashboard combines Claude.ai chat interaction data, O*NET task structure, and BLS labor market signals to show where AI looks more like a productivity tailwind versus a displacement risk.

Summary AI Labor Impact

This tab maps each occupation using two existing scores: the occupation’s AI augmentation-vs.-automation score on the horizontal axis and the labor shortage composite score on the vertical axis.

x-axis = occupation_ai_score
y-axis = labor_shortage_composite_score

The chart itself does not create a new blended score; instead, it visually combines the scoring systems from the other two analytical tabs so you can see where shortage pressure and AI exposure reinforce each other.

How the ranked lists are built

The “benefit” list favors occupations with positive AI scores and high shortage scores, while the “suffer” list favors occupations with negative AI scores and low shortage scores. Rankings are based on combined percentile position across the two plotted axes.

benefit_rank = percentile(ai_score) + percentile(shortage_score)
suffer_rank = (1 - percentile(ai_score)) + (1 - percentile(shortage_score))

That means a job rises into the top-benefit list only if it is strong on both augmentation and shortage pressure, and rises into the top-suffer list only if it is weak on both.

AI Augmentation vs. Automation

This tab shows, for each occupation, whether currently observed Claude usage patterns lean more toward augmenting workers or automating away the work they do.

Occupation score

The occupation score is a weighted average of task scores, where more frequently performed tasks matter more than rare tasks.

occupation_score = Sum(task_score x freq_weight) / Sum(freq_weight)

freq_weight = log(1 + annual_frequency)

O*NET frequency ratings are converted into estimated annual occurrences, then compressed with a log transform so daily work counts more than yearly work without letting ultra-frequent tasks dominate the full occupation score.

Coverage threshold: An occupation needs at least 2 AI-exposed tasks and at least 10.1% task coverage to receive a non-zero AI score, which helps avoid misleading scores from very sparse AI data.

Task score

Each task score depends on whether Claude is being used in more augmentation-like or automation-like ways, and on how central that task is to the occupation.

High-importance tasks (importance > 3.5)

For tasks that define the core of the job, augmentation is positive for the worker and automation is negative.

task_score = 100 x (relevance / 100) x importance_weight x (augmentation_share - automation_share)

Lower-importance tasks (importance ≤ 3.5)

For more peripheral work, offloading the task is generally treated as beneficial because it frees up time for more valuable work.

task_score = 100 x (relevance / 100) x max(automation_share, augmentation_share) x relief

relief = max(0, (3.5 - importance) / 2.5)

Importance scaling

importance_weight = (importance - 1) / 4   [range: 0 to 1]

Importance uses O*NET’s 1–5 scale, while relevance uses O*NET’s 0–100 task relevance score.

Interaction classification and shares

Anthropic’s Economic Index classifies Claude conversations into five interaction types. For this dashboard, Directive and Feedback Loop are grouped as automation, while Task Iteration, Validation, and Learning are grouped as augmentation.

automation_share = (Directive + Feedback Loop) / total
augmentation_share = (Task Iteration + Validation + Learning) / total

total = all classified + unclassified interactions

Unclassified interactions dilute both shares equally rather than favoring one side.

Human-required task filter

Tasks that still fundamentally require an in-person human presence, embodied performance, or physical execution are excluded from AI exposure even when some Claude interactions are observed for them. This keeps the score focused on work that can plausibly be reshaped by current AI usage.

Labor Shortage Scorecard

This tab scores each occupation’s shortage pressure using the workbook’s three-pillar scorecard: projected employment growth, recent wage growth, and recent actual employment growth.

Composite score

composite_score = (projected_employment_score + wage_score + actual_employment_score) / 3

Each pillar is scaled to a 1–5 range using the workbook’s percentile-rank logic, and the dashboard keeps the workbook’s equal one-third weighting across the three pillars.

Projected employment pillar

projected_employment_cagr = (projected_emp_2034 / projected_emp_2024)^(1/10) - 1

projected_employment_score = 1 + 4 x PERCENTRANK.INC(all_projected_cagrs, occupation_projected_cagr)

This uses the BLS 2024–2034 employment projection for the matched occupation. If no projection row is matched, the pillar stays blank.

Wage pillar

wage_cagr_scorecard = use 2019–2024 CAGR if available;
otherwise use 2021–2024 CAGR;
otherwise use 2023–2024 CAGR

wage_score = 1 + 4 x PERCENTRANK.INC(all_wage_cagrs, occupation_wage_cagr)

The dashboard surfaces the underlying wage series so the user can see the actual values behind the score and which fallback time window the workbook used.

Actual employment pillar

employment_cagr_scorecard = use 2019–2024 CAGR if available;
otherwise use 2021–2024 CAGR;
otherwise use 2023–2024 CAGR

actual_employment_score = 1 + 4 x PERCENTRANK.INC(all_employment_cagrs, occupation_employment_cagr)

The drill-down also shows the underlying 2019–2024 employment series and the time window used for the score.

Data Sources

Anthropic Economic Index
Claude.ai consumer chat usage classified by O*NET task and interaction type.
huggingface.co/datasets/Anthropic/EconomicIndex
O*NET Task Ratings
U.S. Department of Labor occupational database with task statements and ratings for importance, relevance, and frequency that are used to weight the AI score.
onetcenter.org/dictionary/20.1/excel/task_ratings.html
BLS OEWS history
Historical occupation-level wage and employment series embedded from the workbook’s scorecard inputs and used for the wage and actual employment pillars.
BLS 2024–2034 projections
Projected 2024 employment, 2034 employment, numeric change, CAGR, and annual openings taken from the workbook’s projection source sheet.

All scores in this dashboard reflect observed current usage patterns in the underlying datasets, not a forecast of ultimate AI capability or guaranteed job outcomes.
